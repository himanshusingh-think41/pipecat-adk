# Turn ID Propagation

## Overview

`turn_id` is a UUID hex string minted by `VqlUserContextAggregator` for each user turn. It travels through the Vql layer so that `VqlAssistantContextAggregator` can correlate the spoken TTS output back to the turn that triggered it — without storing any state in the aggregator itself.

`invocation_id` is the identifier ADK generates internally when `runner.run_async` is called. It never appears in Vql frames; only `AdkLLMService` knows both IDs.

---

## The Propagation Chain

```
VqlUserContextAggregator
  mint turn_id = uuid4().hex
  push VqlContextFrame(turn_id, text) downstream
  store _prev_turn_id = turn_id          ← only state in the aggregator

AdkLLMService
  receive VqlContextFrame
  call runner.run_async(new_message=...)
    learn invocation_id from first event
    store _turn_invocation_map[turn_id] = invocation_id
    push VqlLLMFullResponseStartFrame(turn_id, invocation_id)

VqlTTSMixin (applied to TTS service)
  intercept VqlLLMFullResponseStartFrame
  set self._turn_context_id = frame.turn_id
  TTS now stamps context_id = turn_id on every TTSTextFrame / TTSStoppedFrame

VqlAssistantContextAggregator
  accumulate TTSTextFrame into _aggregation (via parent)
  on TTSStoppedFrame(context_id=turn_id):
    push VqlTurnCompletedFrame(turn_id, text, interrupted=False) UPSTREAM
  on VqlInterruptionFrame(turn_id):
    push VqlTurnCompletedFrame(turn_id, text, interrupted=True) UPSTREAM

AdkLLMService (upstream)
  receive VqlTurnCompletedFrame
  invocation_id = _turn_invocation_map.pop(turn_id)
  if interrupted and text: write [HEARD] event to ADK session
```

---

## Why No Pre-Persist

The previous design pre-persisted the user event to ADK before calling the runner, then resumed via `invocation_id`. This required `ResumabilityConfig(is_resumable=True)` as a hard prerequisite.

The new design uses `runner.run_async(new_message=content)` directly — ADK persists the event and generates `invocation_id` as part of the same call. `ResumabilityConfig` is no longer needed.

**What changed:** `_build_user_event` now returns `Content` (not `Event`), and the session write is implicit in `runner.run_async`. Subclasses that previously built `Event` objects with custom `invocation_id` should migrate to returning `Content` — ADK handles the event wrapping.

---

## Stateless turn_id in VqlAssistantContextAggregator

The assistant aggregator needs to know which turn_id corresponds to the text it just accumulated. It gets this statelessly from two sources:

| Path | Frame | Field |
|------|-------|-------|
| Clean turn | `TTSStoppedFrame` | `context_id` (set to `turn_id` by `VqlTTSMixin`) |
| Interrupted turn | `VqlInterruptionFrame` | `turn_id` (broadcast by `VqlUserContextAggregator`) |

Neither path requires the aggregator to store `turn_id`. This means the aggregator has no `_current_turn_id` or similar state — turn_id lives in the frame.

---

## VqlInterruptionFrame

When the user starts speaking mid-response, `VqlUserContextAggregator.broadcast_interruption()` is called. The override:

1. Calls `_FrameProcessor__reset_process_task()` (name-mangled pipecat private method) to abort in-flight processing
2. Broadcasts `VqlInterruptionFrame(turn_id=_prev_turn_id)` — carrying the turn_id of the turn being interrupted

`VqlInterruptionFrame` extends pipecat's `InterruptionFrame` so all downstream components that react to `InterruptionFrame` still work correctly. The additional `turn_id` field is only read by `VqlAssistantContextAggregator`.

---

## VqlTTSMixin Is Required

`VqlTTSMixin` intercepts `LLMFullResponseStartFrame` in the TTS service's `process_frame`. When the incoming frame is a `VqlLLMFullResponseStartFrame`, it sets `self._turn_context_id = frame.turn_id`. The pipecat TTS base class already reads `_turn_context_id` and stamps it onto `TTSStoppedFrame.context_id`.

Without `VqlTTSMixin`, `TTSStoppedFrame.context_id` is a random UUID generated by the TTS service and the assistant aggregator cannot correlate it to any turn_id. The `VqlTurnCompletedFrame` will never be pushed and `[HEARD]` events will never be written — interruption handling silently degrades.

**Always apply `VqlTTSMixin` to your TTS service:**

```python
class MyGoogleTTSService(VqlTTSMixin, GoogleTTSService):
    pass
```
